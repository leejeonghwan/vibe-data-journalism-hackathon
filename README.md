# 바이브 코딩 x 데이터 저널리즘 해커톤 (슬로우뉴스)

코드를 몰라도 방대한 공공데이터에서 기삿거리를 찾아내는 방법을 다루는 강의 자료. 기본 전제는 **저널리즘 원칙을 잃지 말라**는 것이다. 생성형 인공지능도 스프레드시트처럼 도구일 뿐, 의제 설정과 검증, 기사 작성의 모든 책임은 기자가 진다.

> 정확한 질문이 정확한 데이터를 만든다.

## 구성 (정적 사이트, 빌드 불필요)

여섯 강 + 강의 홈으로 이루어진 멀티페이지 사이트. 모든 페이지가 공용 CSS 하나(`assets/style.css`)와 스크립트 하나(`assets/app.js`)를 공유한다.

```
hackathon-site/
├── index.html              # 강의 홈 (표지·목차)
├── 01-first-exercise.html  # 1강 · 첫 실습 30분 (입문)
├── 02-method.html          # 2강 · 기초취재 방법론 (방법)
├── 03-data-100.html        # 3강 · 공공데이터 100선 (자료)
├── 04-case-colin.html      # 4강 · 콜린알포 케이스 (심화)
├── 05-manual.html          # 5강 · 프로젝트 매뉴얼 (시스템·운영)
├── 06-verification.html    # 6강 · 검증과 원칙 (핵심)
├── assets/
│   ├── style.css           # 공용 디자인
│   └── app.js              # 공용 내비게이션
├── .nojekyll
└── README.md
```

## 권하는 학습 동선

- 해커톤 참가자: **1 → 2 → 3**으로 손을 풀고, **6(검증)**을 반드시 함께 읽는다.
- **4(케이스)**와 **5(매뉴얼)**는 한 단계 더 깊이 들어갈 때의 참고서.

## GitHub Pages로 공개하기

1. 이 폴더(`hackathon-site`)를 깃허브 새 저장소에 올린다.
2. 저장소 **Settings → Pages → Source**를 `Deploy from a branch`로, 브랜치 `main` / 폴더 `/ (root)`로 설정하고 저장.
3. 잠시 후 `https://<아이디>.github.io/<저장소이름>/` 에서 공개된다.

> 코워크 사용자라면 이 폴더를 열고 **"이 폴더를 깃허브에 새 저장소로 올리고 GitHub Pages로 공개해 줘"** 라고 부탁하면 된다. (5강 참고)

`.nojekyll` 파일은 GitHub Pages가 폴더·파일을 그대로 서빙하도록 하는 설정으로, 그대로 둔다.

## 로컬에서 미리 보기

```bash
cd hackathon-site
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000
```
